## Other Files
- `GeoClassChangeTrigger.h` — volume trigger for runtime class switching
- `GeoTeleporter.h` — overlap pad that teleports the touching `APlayableCharacter` to the next `AGeoTeleporter` sharing its `TeleportTag` (same-tag pads sorted by actor name, wrapping — each tag group forms its own cycle). Server does the teleport (`SetActorLocation` with `ETeleportType::TeleportPhysics`, XY of the destination pad, Z kept); the destination's `PendingArrivals` set eats arrival overlaps so they don't chain-teleport — the character stays in the set until they walk off the pad (end overlap), because begin overlap can fire more than once for a single arrival (deferred dispatch + the movement component finishing its move). A separate `ArenaTag` (the arena on the **far side** of the trip — unset = pad never changes arena) is what moves the players between rooms: alongside the move, the server calls `AGeoGameState::SetActiveArena(FindArena(ArenaTag))`, and everything keyed on the active arena (camera bounds, the wipe/checkpoint entrance, deployable push-out) follows from that one write. The pad never touches the camera directly — it used to, locally per client, which framed a client on the arena it had just left. `FindNextTeleporter()` is deterministic and needs no authority, so the destination's `PendingArrivals.Add` runs on **both** client and server (only the `SetActorLocation` move and the arena switch stay server-gated). Both tags are EditAnywhere + BlueprintReadWrite; editor-created tags fine. In `BeginPlay`, each pad rotates (yaw only, via `SetActorRotation` on the flat 2D direction to its next pad) to face where it sends you — the mesh's arrow points at the destination; computed once at level start, not re-evaluated if teleporters move. A `UTextRenderComponent` attached to the mesh shows editor-tweakable `DisplayText` (`OnConstruction` re-syncs it, so edits are visible immediately in-editor and in-game).
- `GeoMatchStateButton.h` — overlap button; requests a configurable match-state transition (`StartMatch` / `WaitingToStart` / `WaitingPostMatch`) from `AGeoGameMode`. Server-only, mirrors `GeoClassChangeTrigger`.
- `GeoInteractableActor.h` — base for world actors that receive `FInteractableActorData`. Key fields: `Owner` (actor whose ASC drives GAS/team checks), `Instigator` (origin avatar for world position/direction — may differ from Owner). Always use `Owner` for ASC lookups; use `Instigator` for cue params and spatial calculations.
- `GeoEffectZone.h` — editor-placeable area that applies a configurable `TArray<TInstancedStruct<FEffectData>>` to actors inside its capsule. Extends `AGeoInteractableActor` (owns its own ASC), so it needs no spawner/ability — configure `Team`, `AttitudeBitmask`, `EffectDataArray`, `Radius`, `Level` in the Details panel. Self-inits GAS in `BeginPlay`; overlap delegates (`OnBeginOverlap`/`OnEndOverlap`) maintain `ActorsInZone`. `Tick()` re-applies heal/damage effects (magnitude treated as a per-second rate) to every tracked actor; other effect types are applied once on entry and removed on exit. Build heal zones (heal effect + Friendly attitude) or damage zones (damage effect + configurable `AttitudeBitmask`) purely from data.
- `GeoArena.h` — **one boss encounter**; base class for every arena in a level. Holds the encounter's `BossClass`, its single `ArenaTag`, and its optional `Barrier`. **Target points are addressed by two tags, never one**: a native `TargetPoint.*` purpose (`BossSpawn`, `Entrance`, `FightLocation`, `FightCenter`, `CameraBounds`, `FallRespawn`, `FiringPoint` — all in `FGeoGameplayTags`) plus the editor-authored `Arena.*` tag naming the encounter. `GeoLib::GetTargetPoints(WorldContext, PurposeTag, ArenaTag)` requires both, so a new arena needs **only its own `Arena.*` tag** — the purposes are shared and live in code. One point can carry several purposes and several arena tags (the shared upper camera corners belong to `Arena.Entrance` + `Arena.Main`). The one trap: purposes and arenas multiply out, so a point carrying two purposes *and* two arena tags claims all four combinations — split it into separate points when that is not what you mean. `AGeoArena::GetArenaOfBoss(Boss)` (`Cast<AGeoArena>(Boss->GetOwner())`) is how anything holding a boss reaches its arena, and therefore its `ArenaTag`. **Exactly one boss per arena** — there is no dummy slot; a training dummy gets its own `AGeoDummyArena` instead. Spawns its boss in `BeginPlay` with `Owner = this` (so `AGeoEnemyAIController::TriggerAggro` resolves the arena off the pawn's owner) and keeps a **replicated `Boss` pointer** — clients need it for the boss health bar. `IsBoss(Actor)` is a pointer compare, not a class compare, and is `virtual` — it is the single gate on the whole match machinery, so an arena that refuses it never *starts a match* (it can still be `ActiveArena`: the hub is).

**The arena owns everything room-shaped; the GameState owns only which arena is active and whether a match runs.** Fight lifecycle: `StartFight()` (barrier closes, `CommitFightTimer` armed from the GameState's global `CommitFightTime`) → `CommitFight()` (teleports players to this arena's `TargetPoint.FightLocation`) → `EndFight()` (barrier opens, both timers cleared). All three are `virtual` — subclasses hook them to arm whatever only makes sense once the fight is really live; the timer dispatches through the vtable, so a `CommitFight` override runs on the timer without any extra plumbing. `ResetBoss()` respawns the boss when it is gone or `ResetForNewAttempt()`s it in place.

**`RespawnPlayer(Player)` is the arena's death policy and the only reason a death reaches an arena at all** — `APlayableCharacter::DeathLogic` calls it on `GetActiveArena()`, and a disconnect arrives the same way because `AGeoGameMode::Logout` calls `Death()` on the leaver's pawn (Logout runs *before* `UnPossess()`, so a leaver still reads as standing otherwise). Base behaviour: do nothing until `AreAllPlayersDead()`, then `RespawnTimer` for `DeathTime` seconds → `RespawnGroup()` — teleport to `TargetPoint.Entrance`, `RevivePlayers()`, `RequestWaitingToStart()` (all three on `AGeoGameState`, which owns the match-state write). A **victory** does not come back here at all: `AGeoGameState::NotifyBossDefeated()` stands the match down directly and leaves players standing where they killed the boss, with the loot shower running until they re-aggro the respawned boss. The wipe check walks live player controllers rather than a snapshot taken at fight start, so late joiners and leavers need no bookkeeping. It also covers dying with no fight running: a lone death is trivially "everyone is down", so the group respawn runs and `RequestWaitingToStart()` no-ops on an unchanged match state. Per-arena tunables that used to be global on the GameState: `DeathTime`, `FightZoneTagName`, `EntranceZoneTagName`. A level holds one arena per encounter and the GameState runs whichever is active, so nothing in the match machinery knows any boss or room by class.
- `GeoHexArena.h` — destructible hexagonal boss platform; an `AGeoArena` whose fight is fought on breakable ground. One ISM instance per tile (~91, pointy-top axial grid, `GridRadius` rings) attached under the inherited root; tiles are **pure visuals with NoCollision** laid over a flat invisible floor: "falling" is game logic. Replicated `TArray<uint8> TileStates` is the source of truth; `ApplyTileVisuals()` diffs against `AppliedTileStates` and hides dead tiles by zero-scaling their instance (**never `RemoveInstance`** — index stability). Server calls `ApplyTileVisuals()` manually after mutating (OnReps don't fire on authority). Server API: `DestroyTiles(TConstArrayView<FIntPoint>)`, `DestroyTilesInRadius(Center, Radius)`, `ResetAllTiles()`; queries: `IsTileAlive`, `GetTileUnderLocation`, `TileToWorld`. Server `Tick` is **one rule — anything standing over a hole falls**, applied to everything inside `FallCheckRadius`: players are killed (`AGeoCharacter::Death()` + teleport of the corpse to the arena's `TargetPoint.FallRespawn` point) **except mid-dash** (`HasRootMotionSources()` — dash jumps over holes), and any `AGeoDeployableBase` is `Recall()`ed. That single rule is what makes "destroy the tile under it" the counterplay to the boss's turrets and mines, and it costs nothing extra — the tile lookup was already there. Don't add a separate registry or a tile-bound deployable base class; the fall check *is* that mechanism.
Boss abilities reach the arena with **`GetArenaOfBoss(Boss)`** (`Cast<AGeoHexArena>(Boss->GetOwner())` — arenas spawn their boss with `Owner = this`), never by class lookup. Tile queries for abilities: `IsOverAliveTile(WorldLocation)`, `GetRandomAliveTiles(Stream, Ring, Count)` (distinct tiles; `Ring < 0` or an emptied ring draws from the whole platform), `GetLastAliveTileAlongRay(Origin, Direction, MaxRange, OutTile)` (walks in half-tile steps so a clipped tile is never skipped, and stops at the far edge of the platform even when the ray outranges it), plus `GetTileRing(Tile)` and `GetGridRadius()` for ring maths. **Tick is the fall check and it is gated on the fight lifecycle**: it starts disabled (`bStartWithTickEnabled = false`), `CommitFight()` enables it, `EndFight()` disables it and resets the tiles. That gate is the whole point — before commit nobody has been teleported onto the platform, so players standing in the void around it (or still walking in) must not be killed. Don't add a separate "is the fight live" flag; the tick enable *is* that state.
- `GeoDummyArena.h` — an `AGeoArena` for a training dummy: it spawns its enemy at level start and then just stands there. Header-only, and the whole class is two one-liners: `IsBoss() → false` and `RespawnPlayer(Player) → Player.Revive()`. `EntranceArena` in `DraftMap` is one: the hub's training dummy, and the hub itself (`Arena.Entrance`, the GameState's `DefaultArenaTag`). `IsBoss` is genuinely sufficient to stay out of the match: `AGeoEnemyAIController::TriggerAggro` is the **only** place that calls `StartMatch()`, and it asks `IsBoss` first — so refusing there means no match, and every downstream effect (boss health bar, `OnEnemyDefeated` → loot binding, barrier, fight commit, camera bounds swap) is reached only through an active match. `RespawnPlayer` is the other half: no wipe to wait for at a dummy, so whoever dies gets straight back up where they fell. That instant rez used to be an accident — `AGeoGameState::NotifyPlayerDied` revived anyone who died while `MatchState != InProgress`, and a dummy arena simply never ran a match — so the hub's respawn behaviour was written in the GameState's match check rather than anywhere you would think to look. Don't add no-op `StartFight`/`CommitFight`/`EndFight` overrides "for safety": they are unreachable on a dummy arena and would be dead code. Only `BossClass` and `ArenaTag` are meaningful on it — `ArenaTag` still matters because its `TargetPoint.BossSpawn` point is resolved through it, and the hub's camera/entrance points hang off the same tag; `Barrier` is never read because the arena never runs a fight. It *is* `AGeoGameState::ActiveArena` at level start (the hub is an arena like any other) — being active is not the same as running a match, which is what `IsBoss` gates.
- `GeoArenaBarrier.h` — replicated barrier that blocks the arena entrance; opens/closes via `SetClosed(bool)` (server-only); animated actors lerp full `FTransform` between `FightOnTransform` and `FightOffTransform` over `CommitFightTime` (read from `AGeoGameState`). Editor helpers: `CaptureFightOn/OffTransforms`, `SetToFightOn/OffTransforms`. The lerp is driven by the actor's own `Tick`, enabled from `OnRep_bIsClosed` (the single state-change seam, called manually by `SetClosed` on the server since OnReps don't fire on the authority) and disabled once `LerpAlpha` settles at its target. Drive it *only* that way — the previous `SetTimerForNextTick` chain had a tautological stop condition (`LerpAlpha` is clamped to `[0,1]`, so `bIsClosed && LerpAlpha <= 1 || !bIsClosed && LerpAlpha >= 0` is always true), so no chain ever terminated and every redundant entry point spawned a parallel one that each advanced the alpha once per frame. The barrier closed N× too fast, N = live chain count, accumulating with every open/close. `OnBarrierStateChanged` is a pure `BlueprintImplementableEvent` for VFX only — never hang lerp driving off it, or a BP override that forgets the parent call freezes the barrier.
