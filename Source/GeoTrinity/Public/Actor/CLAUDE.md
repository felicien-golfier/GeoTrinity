## Other Files
- `GeoClassChangeTrigger.h` — volume trigger for runtime class switching
- `GeoTeleporter.h` — overlap pad that teleports the touching `APlayableCharacter` to the next `AGeoTeleporter` sharing its `TeleportTag` (same-tag pads sorted by actor name, wrapping — each tag group forms its own cycle). Server does the teleport (`SetActorLocation` with `ETeleportType::TeleportPhysics`, XY of the destination pad, Z kept); the destination's `PendingArrivals` set eats arrival overlaps so they don't chain-teleport — the character stays in the set until they walk off the pad (end overlap), because begin overlap can fire more than once for a single arrival (deferred dispatch + the movement component finishing its move). A separate `CameraTag` (the camera zone the pad sits in; unset = pad never touches the camera) drives the camera: any overlap by the locally controlled player (walk-in or replicated arrival) calls `AGeoGameCamera::SetBoundsTag` with it, so the camera reframes to the `AGeoTargetPoint`s carrying that tag. Both tags are EditAnywhere + BlueprintReadWrite; editor-created tags fine. In `BeginPlay`, each pad rotates (yaw only, via `SetActorRotation` on the flat 2D direction to its next pad) to face where it sends you — the mesh's arrow points at the destination; computed once at level start, not re-evaluated if teleporters move. A `UTextRenderComponent` attached to the mesh shows editor-tweakable `DisplayText` (`OnConstruction` re-syncs it, so edits are visible immediately in-editor and in-game).
- `GeoMatchStateButton.h` — overlap button; requests a configurable match-state transition (`StartMatch` / `WaitingToStart` / `WaitingPostMatch`) from `AGeoGameMode`. Server-only, mirrors `GeoClassChangeTrigger`.
- `GeoInteractableActor.h` — base for world actors that receive `FInteractableActorData`. Key fields: `Owner` (actor whose ASC drives GAS/team checks), `Instigator` (origin avatar for world position/direction — may differ from Owner). Always use `Owner` for ASC lookups; use `Instigator` for cue params and spatial calculations.
- `GeoEffectZone.h` — editor-placeable area that applies a configurable `TArray<TInstancedStruct<FEffectData>>` to actors inside its capsule. Extends `AGeoInteractableActor` (owns its own ASC), so it needs no spawner/ability — configure `Team`, `AttitudeBitmask`, `EffectDataArray`, `Radius`, `Level` in the Details panel. Self-inits GAS in `BeginPlay`; overlap delegates (`OnBeginOverlap`/`OnEndOverlap`) maintain `ActorsInZone`. `Tick()` re-applies heal/damage effects (magnitude treated as a per-second rate) to every tracked actor; other effect types are applied once on entry and removed on exit. Build heal zones (heal effect + Friendly attitude) or damage zones (damage effect + configurable `AttitudeBitmask`) purely from data.
- `GeoHexArena.h` — destructible hexagonal boss platform (replicated `AActor`, not an InteractableActor — no ASC). One ISM instance per tile (~91, pointy-top axial grid, `GridRadius` rings), tiles are **pure visuals with NoCollision** laid over a flat invisible floor: "falling" is game logic. Replicated `TArray<uint8> TileStates` is the source of truth; `ApplyTileVisuals()` diffs against `AppliedTileStates` and hides dead tiles by zero-scaling their instance (**never `RemoveInstance`** — index stability). Server calls `ApplyTileVisuals()` manually after mutating (OnReps don't fire on authority). Server API: `DestroyTiles(TConstArrayView<FIntPoint>)`, `DestroyTilesInRadius(Center, Radius)`, `ResetAllTiles()`; queries: `IsTileAlive`, `GetTileUnderLocation`, `TileToWorld`. Server `Tick` kills any player inside `FallCheckRadius` standing over a dead tile or the void (`AGeoCharacter::Death()` + teleport to the `AGeoTargetPoint` tagged `FallRespawnTag`, via `GeoLib::GetTargetPoints` — required: outside InProgress the GameState revives dead players in place), **except mid-dash** (`HasRootMotionSources()` — dash jumps over holes). Owns its boss (`HexBossClass` spawned at center, deliberately outside the GameState single-boss machinery): defeat (`OnEnemyDefeated`) and group wipe (`OnMatchStateChanged` → `WaitingToStart`) reset all tiles and respawn/reset the boss.
- `GeoArenaBarrier.h` — replicated barrier that blocks the arena entrance; opens/closes via `SetClosed(bool)` (server-only); animated actors lerp full `FTransform` between `FightOnTransform` and `FightOffTransform` over `CommitFightTime` (read from `AGeoGameState`). Editor helpers: `CaptureFightOn/OffTransforms`, `SetToFightOn/OffTransforms`. The lerp is driven by the actor's own `Tick`, enabled from `OnRep_bIsClosed` (the single state-change seam, called manually by `SetClosed` on the server since OnReps don't fire on the authority) and disabled once `LerpAlpha` settles at its target. Drive it *only* that way — the previous `SetTimerForNextTick` chain had a tautological stop condition (`LerpAlpha` is clamped to `[0,1]`, so `bIsClosed && LerpAlpha <= 1 || !bIsClosed && LerpAlpha >= 0` is always true), so no chain ever terminated and every redundant entry point spawned a parallel one that each advanced the alpha once per frame. The barrier closed N× too fast, N = live chain count, accumulating with every open/close. `OnBarrierStateChanged` is a pure `BlueprintImplementableEvent` for VFX only — never hang lerp driving off it, or a BP override that forgets the parent call freezes the barrier.
