# Actor

World actors: projectiles, deployables, turret, triggers, arenas.

## Subfolders
| Folder | Contents |
|---|---|
| `Projectile/` | `GeoProjectile` base + `GeoPooledProjectile`, `GeoShieldBurstProjectile`, `DeployableSpawnerProjectile` |
| `Deployable/` | `GeoDeployableBase` + `GeoWall`, `GeoHealingZone`, `GeoBuffPickup` |
| `Turret/` | `GeoTurret` — Triangle's deployable turret |

## Other Files
- `GeoClassChangeTrigger.h` — volume trigger for runtime class switching.
- `GeoTeleporter.h` — overlap pad that teleports the touching `APlayableCharacter` to the next same-`TeleportTag` pad (name-sorted, wrapping cycle per tag). Server-only move (`SetActorLocation`, XY only); `PendingArrivals` set on the destination absorbs repeat begin-overlap events until the character walks off (end overlap) — needed because begin-overlap can fire more than once per arrival. Pure position move: no arena/GameState write. `PendingArrivals.Add` runs on both client and server (deterministic); only the actual move is server-gated. Pads rotate once in `BeginPlay` to face their destination.
- `GeoMatchStateButton.h` — overlap button requesting a match-state transition from `AGeoGameMode`. Server-only.
- `GeoInteractableActor.h` — base for actors receiving `FInteractableActorData`. `Owner` = ASC/GAS/team-check source; `Instigator` = origin avatar for cue params/spatial calcs — they may differ (e.g. boss minions). Always use `Owner` for ASC lookups.
- `GeoEffectZone.h` — editor-placeable capsule applying `EffectDataArray` to actors inside. Owns its own ASC (extends `AGeoInteractableActor`), so no spawner needed — pure data config. `Tick()` re-applies heal/damage each frame (magnitude = per-second rate); other effect types apply once on entry/exit.
- `GeoArena.h` — one boss encounter, self-driving off match state (no GameState arena pointer). Target points use two tags: a shared `TargetPoint.*` purpose + the arena's own `Arena.*` tag (`GeoLib::GetTargetPoints` needs both) — a new arena only needs its own `Arena.*` tag. `GetArenaOfBoss(Boss)` = `Cast<AGeoArena>(Boss->GetOwner())`, the standard way abilities reach their arena. Exactly one boss per arena (no dummy slot — use `AGeoDummyArena`). Replicated `Boss` pointer for the client health bar. `IsBoss()` is a pointer compare, `virtual`, and the sole gate on match start.
  - Fight lifecycle driven by replicated `bFighting` (not a GameState pointer): `StartFight()` (called by `AGeoEnemyAIController::TriggerAggro` on aggro) sets `bFighting`, shows boss bar, sets GameState checkpoint, closes barrier, arms `CommitFightTimer`; refuses if a fight is already running. `CommitFight()` teleports players to `TargetPoint.FightLocation` (skips those already in `FightZoneTagName`). `EndFight()` (via `AGeoGameState::OnMatchStateChanged`, server-only) clears timer, drops `bFighting`, hides bar, resets boss — reached on both wipe and victory; entering InProgress instead calls `StopLoot()` on every arena. Both are `virtual` for subclass hooks.
  - Loot: `OnBossDefeated` starts a looping shower (`SpawnLootBurst`) that runs until any fight starts anywhere (`StopLoot()` on `OnMatchStateChanged` entering InProgress). `GetFightingArena()` is the static "who's fighting" query.
  - Player death is entirely `AGeoGameState`'s policy (`NotifyPlayerDied`), not the arena's — the arena's only role is registering its `ArenaTag` as the respawn checkpoint in `StartFight()`. See `GameClasses/CLAUDE.md`.
- `GeoHexArena.h` — `AGeoArena` on a destructible hex tile grid. Tiles are pure visuals (NoCollision) over a flat invisible floor; replicated `TArray<uint8> TileStates` is truth, `ApplyTileVisuals()` diffs and zero-scales dead tiles (never `RemoveInstance` — index stability). Server calls `ApplyTileVisuals()` manually after mutating (OnReps don't fire on authority). Server Tick = one rule: anything over a hole falls (players die + respawn at `TargetPoint.FallRespawn`, except mid-dash; deployables `Recall()`). Tick is gated on fight lifecycle (`CommitFight` enables, `EndFight` disables+resets) so nobody dies standing in the pre-fight void. Reach the arena via `GetArenaOfBoss` (never class lookup).
- `GeoDummyArena.h` — training-dummy arena: spawns its enemy and does nothing else (`IsBoss() → false`). Since `TriggerAggro` checks `IsBoss` before calling `StartMatch()`, refusing there means the whole fight machinery (bar, loot, barrier, commit) never fires. No `RespawnPlayer` override needed — death here is always "out of fight", handled generically by `AGeoGameState`.
- `GeoArenaBarrier.h` — replicated barrier; `SetClosed(bool)` (server-only) lerps `FTransform` between Fight-On/Off over `CommitFightTime`/`DeathTime` (from GameState). Lerp is driven by the actor's own `Tick`, enabled only from `OnRep_bIsClosed` (called manually by `SetClosed` on server since OnReps don't fire on authority) — do not drive it any other way (a prior `SetTimerForNextTick` chain had a non-terminating condition and stacked parallel lerps). `OnBarrierStateChanged` is VFX-only, never lerp logic.
- `GeoHexBarrier.h` — `AGeoArenaBarrier` variant: blocks with a hex tile alley instead of moving actors, hiding tiles one by one as `LerpAlpha` climbs (same zero-scale trick as `AGeoHexArena`). No replication of its own — piggybacks on the inherited `bIsClosed`. `bVanishAlongColumns` picks sweep direction. Purely visual — if players should fall in the gap, the alley must sit inside the arena's `FallCheckRadius`.

See each subfolder's `CLAUDE.md` for detail.
