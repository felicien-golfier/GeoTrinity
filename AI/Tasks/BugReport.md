`GetOriginalInstigatorAbilitySystemComponent()` returns null when a GE is applied without an explicit instigator ASC (e.g. the enemy reset-to-full-life path in `EnemyCharacter::OnHealthChanged_Implementation`). The fallback block reaches the dereference and crashes.

**Fix:** Null-check the result before calling `GetOwnerActor()`:
```cpp
UAbilitySystemComponent* InstigatorASC = Data.EffectSpec.GetEffectContext().GetOriginalInstigatorAbilitySystemComponent();
if (InstigatorASC)
{
    SourcePlayerState = Cast<AGeoPlayerState>(InstigatorASC->GetOwnerActor());
}
